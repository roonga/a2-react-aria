A wrapper around React Aria's `Button` component. Handles keyboard, mouse, and touch
activation with full accessibility support.

## Add via CLI

```bash
npx @a2ra/cli add button
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `variant` | `primary`<br />`secondary` <span className="prop-default-badge">default</span><br />`danger`<br />`ghost` | Visual style |
| `size` | `sm`<br />`md` <span className="prop-default-badge">default</span><br />`lg` | Button size |
| `isDisabled` | `true`<br />`false` <span className="prop-default-badge">default</span> | Prevents interaction and applies disabled styles |
| `isPending` | `true`<br />`false` <span className="prop-default-badge">default</span> | Shows a loading indicator; disables the button |
| `type` | `button` <span className="prop-default-badge">default</span><br />`reset`<br />`submit` | HTML button type |
| `name` | `string` | Form field name |
| `value` | `string` | Form field value |
| `children` | `string`<br />`node[]` | Button label text or child nodes |

## Notes

- Use `type: "submit"` inside a `Form` node to trigger form submission.
- `isPending` disables the button and shows a spinner; use it while an async action is running.
- `variant: "ghost"` renders no background or border; suitable for icon buttons inside toolbars.
