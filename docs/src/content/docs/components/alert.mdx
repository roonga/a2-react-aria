A non-interactive status banner with an optional title. Uses `role="alert"` so screen readers
announce it immediately when it appears.

## Add via CLI

```bash
npx @a2ra/cli add alert
```

## Variants

<ComponentPreview client:only="react" node={{
  type: "Flex",
  props: { direction: "column", gap: "sm" },
  children: [

## Example JSON

```json
{
  "type": "Alert",
  "props": {
    "variant": "success",
    "title": "Payment received"
  },
  "children": "Your order has been confirmed."
}
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `variant` | `info` <span className="prop-default-badge">default</span><br />`success`<br />`warning`<br />`error` | Color scheme and semantic meaning |
| `title` | `string` | Bold heading rendered above the body |
| `children` | `string`<br />`node[]` | Body content |

## Notes

- The element carries `role="alert"`, which triggers an ARIA live-region announcement on mount.
  Only render it when the message is genuinely new information; avoid re-rendering the same alert on every tick.
- Choose the variant that matches the semantic meaning, not just the desired colour.
