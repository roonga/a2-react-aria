A wrapper around React Aria's `Switch` component. Provides keyboard, pointer, and touch
interaction with proper ARIA role and state management.

## Add via CLI

```bash
npx @a2ra/cli add switch
```

## Basic

<ComponentPreview client:only="react" node={{
  type: "Flex",
  props: { gap: "md", direction: "column" },

    { type: "Switch", props: { label: "Email notifications" } },
    { type: "Switch", props: { label: "Dark mode", defaultSelected: true } },

## Example JSON

```json
{
  "type": "Switch",
  "props": {
    "label": "Dark mode",
    "name": "darkMode",
    "value": "on",
    "defaultSelected": false
  }
}
```

## Props

| Prop | Type | Description |
|------|------|-------------|
| `label` | `string` | Visible label text |
| `isSelected` | `boolean` | Controlled on/off state |
| `defaultSelected` | `true`<br />`false` <span className="prop-default-badge">default</span> | Uncontrolled default |
| `isDisabled` | `true`<br />`false` <span className="prop-default-badge">default</span> | Disables the switch |
| `isReadOnly` | `true`<br />`false` <span className="prop-default-badge">default</span> | Prevents toggling |
| `isRequired` | `true`<br />`false` <span className="prop-default-badge">default</span> | Marks as required |
| `isInvalid` | `true`<br />`false` <span className="prop-default-badge">default</span> | Shows error styling |
| `validationBehavior` | `aria` <span className="prop-default-badge">default</span><br />`native` | Validation strategy |
| `name` | `string` | Form field name |
| `value` | `string` | Value submitted when selected |
| `description` | `string` | Helper text shown below the label |
| `errorMessage` | `string` | Error text when `isInvalid` is true |

## Notes

- `Switch` uses `role="switch"` so screen readers announce it as a toggle, not a checkbox.
- `value` is the string submitted with the form when the switch is on (like a checkbox `value`).
