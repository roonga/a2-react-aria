A wrapper around React Aria's `TextField` component. Manages label association,
validation state, and error messages automatically.

## Add via CLI

```bash
npx @a2ra/cli add text-field
```

## Basic

<ComponentPreview client:only="react" node={{
  type: "TextField",
  props: {
    label: "Email address",
    type: "email",
    name: "email",
    isRequired: true,
    autoComplete: "email",
    description: "We'll never share your email."
  }
}} />

## Props

| Prop | Type | Description |
|------|------|-------------|
| `label` | `string` | Visible label (required for accessibility) |
| `placeholder` | `string` | Placeholder text |
| `type` | `text` <span className="prop-default-badge">default</span><br />`email`<br />`password`<br />`number`<br />`tel`<br />`url` | Input type |
| `name` | `string` | Form field name |
| `value` | `string` | Controlled value |
| `defaultValue` | `string` | Uncontrolled default |
| `autoComplete` | `string` | Browser autocomplete hint |
| `inputMode` | `text`<br />`numeric`<br />`decimal`<br />`email`<br />`tel`<br />`url`<br />`search` | Virtual keyboard hint |
| `minLength` | `number` | Minimum character length |
| `maxLength` | `number` | Maximum character length |
| `pattern` | `string` | Regex validation pattern |
| `isDisabled` | `true`<br />`false` <span className="prop-default-badge">default</span> | Disables the input |
| `isRequired` | `true`<br />`false` <span className="prop-default-badge">default</span> | Marks field as required |
| `isReadOnly` | `true`<br />`false` <span className="prop-default-badge">default</span> | Prevents editing |
| `isInvalid` | `true`<br />`false` <span className="prop-default-badge">default</span> | Shows error styling |
| `autoFocus` | `true`<br />`false` <span className="prop-default-badge">default</span> | Focuses the field on mount |
| `validationBehavior` | `aria` <span className="prop-default-badge">default</span><br />`native` | Validation reporting strategy |
| `description` | `string` | Helper text shown below the input |
| `errorMessage` | `string` | Error text shown when `isInvalid` is true |

## Notes

- Always provide a `label`. React Aria associates it with the input via ARIA attributes.
- `errorMessage` is only shown when `isInvalid: true`.
- Use `validationBehavior: "native"` for HTML form constraint validation with browser-native messages.
